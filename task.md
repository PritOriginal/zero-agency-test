## В чем суть задачи?

Представь, что мы пишем Telegram-бота для компании. Пользователи пишут туда самые разные вещи: кто-то просто здоровается, кто-то просит найти инструкцию, а кто-то ругается, что у него списали деньги.

Если писать логику на обычных `if/else` или регулярных выражениях, код быстро превратится в кашу, потому что люди формулируют мысли по-разному. 

Поэтому мы сделаем **«Умного диспетчера»**.
Идея простая:
1. Пользователь пишет текст.
2. Наша Go-программа отправляет этот текст в ИИ.
3. ИИ выступает в роли "секретаря-сортировщика". Он читает текст и возвращает нам четкий тег (например: `support` или `chat`).
4. Наша Go-программа читает этот тег и запускает **только тот кусок кода**, который отвечает за эту задачу.

Всё! Тебе нужно написать этот механизм маршрутизации.

## Из каких частей должна состоять программа

Программу нужно разделить на 3 логические части (компонента).

### Часть 1. Навыки (Skills)
Навык — это конкретное действие, которое умеет делать наш бот.
Тебе нужно создать единый интерфейс (правило) для всех навыков. Например:
```go
type Skill interface {
    Execute(userInput string) (string, error)
}
```

Для задания нужно написать 4 простых навыка-заглушки, которые реализуют этот интерфейс:
1. `KnowledgeSkill` (для поиска инфы) — просто выводит в консоль *"Делаю поиск по базе..."* и возвращает текст *"Вот что я нашел в документации: ..."*.
2. `EscalationSkill` (для жалоб) — выводит в консоль *"Зову человека!"* и возвращает *"Переключаю на оператора"*.
3. `ChatSkill` (для болтовни) — возвращает *"Привет! Я ИИ-помощник, чем могу помочь?"*.
4. `FeedbackSkill` (для отзывов) — выводит *"Сохраняю в базу данных..."* и возвращает *"Спасибо за ваш отзыв!"*.

*Важно: Они не должны реально ходить в базу данных, достаточно просто делать `fmt.Println` (имитировать бурную деятельность) и возвращать строку.*

### Часть 2. Классификатор (Связь с ИИ)
Это функция или структура, которая берет текст пользователя, делает HTTP-запрос к ИИ и получает в ответ тег.

Мы дадим тебе доступы только к базовой языковой модели. Сама по себе она ничего не знает о наших тегах. Твоя задача — написать для неё такую инструкцию (запрос), чтобы она проанализировала текст пользователя и вернула один из 4-х тегов:

info_request (передадим в KnowledgeSkill)
support_urgency (передадим в EscalationSkill)
chat (передадим в ChatSkill)
feedback (передадим в FeedbackSkill)

### Часть 3. Роутер (Диспетчер)
Главный компонент программы. Как он должен работать:
1. Роутер хранит в себе map, где тегу соответствует конкретный навык. Примерно так: `map[string]Skill`.
2. Он принимает текст пользователя.
3. Вызывает Классификатор -> получает тег (например, `chat`).
4. Берет из мапы нужный `Skill` по тегу `chat`.
5. Вызывает у этого навыка метод `.Execute()`.
6. Отдает результат.

## Как общаться с ИИ в коде?

Мы даем тебе доступ к нашей модели `gpt-oss-20b`. Она работает по стандарту OpenAI, поэтому общение с ней — это обычный POST HTTP-запрос.

**Вводные данные для подключения:**
- URL: `чуть_позже_допишу/v1/chat/completions`
- Токен (API Key): `[чуть_позже_допишу]`

Чтобы ИИ ответил не просто текстом, а строго тегом, ему нужно задать **Системный промпт**.
Пример тела HTTP-запроса (JSON), который твоя Go-программа должна отправить на сервер ИИ:

```json
{
  "model": "gpt-oss-20b",
  "messages": [
    {
      "role": "system",
      "content": "Ты диспетчер. Прочитай сообщение и верни ТОЛЬКО JSON формата {\"category\": \"название_тега\"}. Допустимые теги: info_request, support_urgency, chat, feedback. Никакого другого текста писать нельзя."
    },
    {
      "role": "user",
      "content": "У меня не проходит оплата, помогите срочно!"
    }
  ]
}
```

Если ИИ послушный, он вернет ответ, в котором будет строка:
`{"category": "support_urgency"}`

Твоя задача — распарсить этот ответ (с помощью `json.Unmarshal`) и достать оттуда значение `support_urgency`.

⚠️ **Подводный камень:** Иногда ИИ слишком умничает и оборачивает JSON в символы маркдауна, возвращая строку вида:
```json
\```
{"category": "support_urgency"}
\```
```

Нужно разобраться, как заставить модель возвращать всегда JSON

## Что программа должна делать при запуске?

Сделай так, чтобы при запуске программы она работала в консоли (читала ввод от пользователя).

**Пример использования:**
```text
> Введите сообщение: Привет, как дела?
[ЛОГ] ИИ определил категорию: chat
[ЛОГ] Запущен навык: ChatSkill
Бот: Привет! Я ИИ-помощник, чем могу помочь?

> Введите сообщение: Верните мои деньги!!
[ЛОГ] ИИ определил категорию: support_urgency
[ЛОГ] Запущен навык: EscalationSkill
[ЛОГ] Зову человека!
Бот: Переключаю на оператора
```
